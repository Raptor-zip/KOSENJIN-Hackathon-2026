# Discord Webhook 連携ガイド

NEMUNAI は居眠り検知・スクワット完了時に、カメラのスクリーンショット付きで Discord に自動通知を送れます。
サーバー不要 — ブラウザから直接 Discord Webhook へ送信します。

## 1. Discord Webhook の作成手順

### 1-1. テキストチャンネルの設定を開く

1. Discord で通知を受け取りたい **サーバー** を開く
2. 対象の **テキストチャンネル** の横にある ⚙ (チャンネルの編集) をクリック

### 1-2. Webhook を作成

1. 左メニューから **連携サービス** → **ウェブフック** を選択
2. **「新しいウェブフック」** をクリック
3. 名前を任意で変更 (例: `NEMUNAI Bot`)
4. アイコンも好みで設定可能

### 1-3. URL をコピー

1. 作成した Webhook の **「ウェブフック URL をコピー」** をクリック
2. 以下のような形式の URL が取得できる:

```
https://discord.com/api/webhooks/1234567890/abcDEFghiJKLmnoPQRstuvWXyz...
```

> **注意:** この URL を知っている人は誰でもメッセージを送信できます。第三者に共有しないでください。

## 2. NEMUNAI での設定

1. NEMUNAI のスタート画面を開く
2. **右上の ⚙ (歯車アイコン)** をクリック
3. 設定モーダルが開くので、**Webhook URL** 欄に先ほどコピーした URL を貼り付ける
4. **「保存」** をクリック

URL は端末の localStorage に保存されるため、次回以降は入力不要です。

## 3. 通知のタイミング

| イベント | メッセージ | 添付 |
|---------|-----------|------|
| 居眠り検知 (monitoring → penalty) | `居眠りを検知しました！` | カメラのスクリーンショット (JPEG) |
| スクワット完了 (penalty → monitoring) | `スクワット5回完了！` | カメラのスクリーンショット (JPEG) |

## 4. 通知を無効にする

設定モーダルで URL 欄を空にして保存すれば、通知は送信されません。

## 5. トラブルシューティング

| 症状 | 対処 |
|-----|------|
| 通知が来ない | URL が正しい形式か確認 (`https://discord.com/api/webhooks/...`) |
| 画像が添付されない | カメラ映像が読み込み済みか確認 (映像が表示されていれば OK) |
| CORS エラー | Discord Webhook はブラウザからの直接 POST を許可しています。プロキシ不要です |
| URL を忘れた | Discord のチャンネル設定 → 連携サービス → ウェブフックから再コピー可能 |
